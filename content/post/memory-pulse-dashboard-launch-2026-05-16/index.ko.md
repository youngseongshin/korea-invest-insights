## 1. 왜 별도의 대시보드를 만들었나

### 1.1 블로그와 대시보드는 서로 다른 도구다

블로그 글은 **'왜 그런가'**를 설명한다. [삼성전자 Citi 목표가 46만원 글](/ko/post/samsung-electronics-citi-tp-460000-memory-rerating-2026-05-11/)이 '메모리 사이클 프레임이 깨질 수 있는가'를 분석했다면, 그건 분기·주간 단위의 깊이 있는 분석이다. 한 번 쓰면 시간이 지나도 논거 자체는 유지된다.

대시보드는 **'지금 이 순간'**을 보여준다. 분석한 논거가 오늘 장에서 어떻게 검증되고 있는지, 어떤 신호가 올라오고 어떤 신호가 사라지고 있는지를 분 단위로 본다.

```
블로그 분석:   "왜 메모리 사이클 프레임이 깨질 수 있는가" (분기·주간 단위, 논거 중심)
Memory Pulse:  "지금 그 논거가 실제 데이터에서 어떻게 작동하는가" (분 단위, 데이터 중심)

→ 두 도구는 보완 관계
→ 분석 글을 읽고 대시보드를 열면, 논거의 현재 상태를 즉시 확인할 수 있다
```

### 1.2 왜 삼성·SK하이닉스 두 종목에 집중하나

코스피 시가총액의 상당 부분이 이 두 회사다. KOSPI 인덱스의 움직임 중 상당량이 이 두 종목으로 설명된다. AI 메모리 사이클의 한국 측 노출도 이 두 회사가 핵심이다.

## 2. 대시보드 구조 — 한 화면에 무엇이 들어가는가

### 2.1 7개 섹션

| 섹션 | 무엇을 보여주나 |
|---|---|
| 1. **Executive Summary** | 장 중 한 줄 요약. 현재 시장이 어떤 톤인지 즉시 파악 |
| 2. **Market Strip** | 코스피·KOSDAQ·환율·SOX 등 핵심 지표 스냅샷 |
| 3. **Source Health Strip** | 각 데이터 소스의 정상 작동 여부 |
| 4. **Company Signals** | 삼성전자·SK하이닉스 카드 (각 회사별 상세 시그널) |
| 5. **Driver Categories** | 시그널을 움직이는 동력의 카테고리 — 미국 반도체 / 환율 / 매크로 / 메모리 사이클 |
| 6. **Instrument Moves** | 종목별 변동표 (Korea / US / Memory / Macro·Risk 필터) |
| 7. **Footer** | 블로그·텔레그램·서브스택 연결 |

### 2.2 데이터 신뢰도 라벨 — 가장 중요한 특징

모든 데이터 행에 **신뢰도 라벨**이 붙어 있다.

```
Live      : 실시간 데이터 (1분 이내)
Delayed   : 지연 데이터 (15분 / 1시간 등 명시)
Fallback  : 대체값 (전일 종가 등)
Missing   : 데이터 누락 (소스 장애)
```

**이게 왜 중요한가**: 시장 데이터는 출처마다 지연이 다르다. 어떤 가격은 실시간, 어떤 가격은 전일 종가, 어떤 가격은 일시 누락이다. 라벨이 없으면 사용자는 데이터가 다 같은 신뢰도라고 착각한다. Memory Pulse는 **모든 데이터의 신뢰도를 투명하게 보여줘서, 사용자가 의사결정 전 데이터 자체를 검증할 수 있게** 한다.

### 2.3 4가지 필터 — 관점에 따라 보기

종목 변동표는 4가지 필터로 분류된다.

| 필터 | 어떤 관점에서 보는가 |
|---|---|
| **Korea** | 코스피·KOSDAQ 종목 — 국내 시장 흐름 |
| **US** | 엔비디아·마이크론·SOX 등 — 미국 반도체 read-through |
| **Memory** | HBM·DDR·NAND 가격 프록시 — 사이클 직접 확인 |
| **Macro / Risk** | 환율·유가·금리·VIX — 매크로 환경 |

**어떻게 활용하는가**: 예를 들어 장 시작 전 'Korea' 탭으로 전일 코스피 마감과 시간 외 거래를 확인하고, 'US' 탭으로 미국 시장의 메모리·반도체 흐름을 본 뒤, 'Macro / Risk' 탭으로 환율·VIX의 시장 환경을 확인한다. 세 가지가 한 화면에 모이면 '오늘 장이 왜 이렇게 시작하는가'에 대한 1차 가설이 잡힌다.

### 4.2 분석 → 대시보드 → 분석의 순환

```
1단계: 블로그에서 새 분석 글 발행
       예) "삼성전자 Citi 목표가 46만원 — 메모리 사이클 프레임 재해석"
       → 논거 정립

2단계: 대시보드에서 실시간 검증
       → 메모리 가격 동향, HBM4E 관련 뉴스, 환율, 외국인 수급 추적
       → 논거의 현재 상태 확인

3단계: 검증 결과로 새 분석 글 작성
       예) "1분기 실적 확인 — 논거 검증" / "추적 변수 변화 — 논거 조정"
       → 분석 업데이트

→ 분석과 데이터가 순환하면서 논거가 다듬어진다
```

### 4.3 두 가지 한계 — 정직하게 짚어둘 점

**한계 1: 대시보드는 매수·매도 신호를 주지 않는다.** 푸터에 명시된 대로 "교육적 시장 정보일 뿐 투자 조언이 아니며, 매수·매도·포지션 사이즈 지시를 내리지 않는다." 시그널을 어떻게 해석할지는 사용자의 몫이다.

**한계 2: 데이터 소스에 의존한다.** Fallback이나 Missing 라벨이 자주 뜨는 시간대(특정 미국 데이터 공급사 장애 등)에는 대시보드의 정확도가 떨어질 수 있다. 그래서 라벨을 명시적으로 보여준다.

**Q: 블로그 분석 글을 읽기 전에 대시보드를 봐도 되나요?**
A: 가능합니다. 대시보드는 독립적으로도 사용 가능합니다. 다만 시그널의 '맥락'을 알면 더 빠르게 해석되므로, [HBM 허브](/ko/page/korea-semiconductor-hbm-kospi-hub/) 또는 최근 분석 글을 먼저 한 번 훑고 대시보드를 보는 게 효율적입니다.

**Q: 대시보드 데이터를 그대로 인용·재배포해도 되나요?**
